# quota-pilot

**Планирование задач с учётом квоты для Claude Code.** Длинные задачи больше не разбиваются о стену 5-часового rate limit: до исчерпания окна сессия оценивает оставшуюся работу, пишет чекпоинт, ставит себе будильник по настенным часам, простаивает с нулевой стоимостью токенов и автоматически продолжает после сброса.

## Почему проактивно?

Существующие инструменты (claude-auto-retry и подобные) **реактивны**: они дают сессии умереть на rate limit, а затем вслепую печатают «continue» через tmux. Тройной провал — обрыв обычно приходится на середину хода (правка сделана, тесты не запущены), поэтому возобновлённая модель неверно судит о том, что реально готово; нужна инъекция клавиш tmux в мёртвую сессию; и ноль предвидения оставшегося бюджета.

quota-pilot переворачивает логику: **сессия никогда не умирает**. Она получает предупреждение *до* исчерпания, сама решает, помещается ли следующая неделимая единица работы в остаток, честно архивирует (включая то, что наполовину сделано и не проверено), и сама себя будит. Без tmux, без launchd, без внешней няньки — только нативные примитивы Claude Code.

## Как это работает

```
┌─ сэмплирование ───────────┐   ┌─ решение ────────────┐   ┌─ поведение (модель) ─────┐
│ основной: опрос oauth/    │   │ хук PostToolUse      │   │ скилл quota-pilot        │
│ usage (троттлинг в хуке,  │ → │ читает state.json    │ → │ 1. оценить единицу       │
│ работает во ВСЕХ сессиях) │   │ порог + кулдаун      │   │ 2. записать чекпоинт     │
│ вспом.: statusline-обёртка│   │ инъекция алерта      │   │ 3. будильник по часам    │
│ (только TUI-дисплей)      │   └──────────────────────┘   │ 4. простой→пробуждение   │
└───────────────────────────┘                              └──────────────────────────┘
```

- **warn** (по умолчанию 88 %): модель оценивает, помещается ли следующая неделимая единица в оставшийся бюджет (резерв 3 % на чекпоинт). Если да — продолжает работать; если нет — архивирует и паркуется.
- **critical** (по умолчанию 95 %): оценка пропускается, немедленное архивирование.
- Чекпоинт (`<проект>/.claude/quota-checkpoint.md`) разделяет *сделано-и-проверено* и *в-работе-не-проверено* — именно это различие устраняет баги слепого возобновления.
- Будильник — цикл по настенным часам, а не один длинный `sleep`: монотонные часы macOS останавливаются во время сна системы, поэтому `sleep 4h` на закрытом ноутбуке просыпает на часы. Цикл обнаруживает пропущенный дедлайн в течение 60 с после пробуждения машины.

## Установка

**Вариант A — установочный скрипт:**

```bash
git clone https://github.com/easyfan/quota-pilot.git
cd quota-pilot
./install.sh                # хук + скилл + команда /quota
./install.sh --statusline   # плюс TUI-дисплей квоты
```

`--statusline` сохраняет существующую statusLine: исходная команда продолжает отрисовываться через обёртку, пока данные квоты захватываются параллельно.

**Вариант B — маркетплейс плагинов:**

```
/plugin marketplace add easyfan/quota-pilot
/plugin install quota-pilot@quota-pilot
```

Удаление: `./install.sh --uninstall` (восстанавливает исходные statusLine и settings; состояние `~/.claude/quota-pilot/` сохраняется, удалите вручную при желании).

## Использование

Делать ничего не нужно — хук следит за квотой при каждом вызове инструмента (после троттлинга максимум один HTTPS-запрос в 60 с). При срабатывании алерта вы увидите, как модель оценивает, архивирует и паркует себя.

- `/quota` — текущая загрузка окон 5ч/7д, обратный отсчёт до сброса, burn rate, прогноз исчерпания
- `touch ~/.claude/quota-pilot/cancel` — разбудить запаркованную сессию раньше срока
- Чекпоинт лежит в `<проект>/.claude/quota-checkpoint.md`; если процесс умер во время парковки, свежая сессия может продолжить с него

## Конфигурация (`~/.claude/quota-pilot/config.json`)

| Ключ | По умолчанию | Значение |
|------|--------------|----------|
| `warn_threshold` | 88 | порог алерта оценки (окно 5ч, %) |
| `critical_threshold` | 95 | порог немедленного архивирования |
| `reserve` | 3 | бюджет, зарезервированный под чекпоинт (%) |
| `cooldown_minutes` | 10 | кулдаун повторного алерта на уровень |
| `max_wait_hours` | 6 | сверх этого — уведомить человека, а не ждать |
| `wake_jitter_minutes` | 5 | случайный джиттер пробуждения (защита от толпы сессий) |
| `seven_day_warn` | 90 | порог уведомления окна 7 дней (только уведомление) |

## Границы

- **Только подписочные аккаунты (Pro/Max).** У аккаунтов с API-ключом нет окон квоты; плагин это определяет и остаётся спящим — ноль накладных расходов, ноль шума.
- Основной сэмплер использует недокументированный эндпоинт `oauth/usage`; каждый ответ проходит валидацию схемы, любое расхождение — тишина, никогда не ложный алерт.
- Квота действует на уровне аккаунта: держите ≤2 одновременно запаркованных длинных задач (джиттер пробуждения предотвращает толпу, но окно всё равно общее).
- Если исчерпано окно 7 дней, сброс 5ч не поможет; ожидания дольше `max_wait_hours` уведомляют вас и останавливаются, а не ждут днями.
